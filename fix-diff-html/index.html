序列化 HTML

序列化HTML有很多中形式：

  1. 根据空格，<>以及其他分割符，把HTML分割成一段段，如把<div class="test">test</div>解析成['<', 'div', 'class="test"', '>', 'test', '<', '/div', '>']这类结构。
  2. 也可以把信息都完整解析出来，如类型，内容，层级关系等，类似DOM树，如上面这个例子可以解析成:

上面这两种都是可行的解析方法，但针对我们的需求来说，第一种优点是比较简单，容易实现，缺点也比较明显，单单拿到其中某一个token，并不能判断出他是属于到底是内容，还是属性。第二种则是多了很多不必要的信息，如层级关系，标签属性顺序等，且跨层级的操作很不方便，对于不符合要求的标签, 如<p<ins>>， 很难体现出来。
